# Proposers

Proposers are responsible for suggesting mathematical templates for the correction term $\Delta$. ADCD supports three different regimes:

---

## 1. Mock Proposer (`proposer="mock"`)
The Mock Proposer is a template-assisted proposer used for baseline testing and ablation studies.
- It contains a library of common mathematical shapes (polynomials, power-laws, exponentials, rationals, and transcendental forms).
- It generates a large pool of candidate templates by combining available variables, constants, and parameters $\theta$.
- **Use case**: Reproducing paper benchmarks, analyzing gate efficiency, and running fast local tests.

---

## 2. Gemini Proposer (`proposer="gemini"`)
The Gemini Proposer uses the Gemini API to query a large language model for physical correction templates.
- It passes the classical law description, variable dimensions, and statistical features of the residual (monotonicity, curvature, oscillation) to the LLM.
- It queries the LLM to suggest physically plausible correction templates using its scientific knowledge.
- **Use case**: Zero-shot discovery in complex scenarios where the correct formula template is unknown.

---

## 3. Hybrid Proposer (`proposer="hybrid"`)
The Hybrid Proposer combines the data-driven proposal strength of the Mock Proposer with the physical reasoning of the LLM.
- Uses the LLM to suggest structural shapes and physical templates.
- Mixes these suggestions with standard mathematical variations generated by the Mock Proposer.
- Passes the combined pool through the physics gates.
- **Use case**: General-purpose scientific discovery.
